# Workshop Testing Facilitator Guide

## Quick Reference

### Testing Schedule
- **Session 1 (Day 429)**: Technical Testing (2 hours)
- **Session 2 (Day 430)**: Full Workshop Dry Run (3 hours)

### Key Testing Areas
1. Environment setup and troubleshooting
2. Content clarity and pacing  
3. Exercise effectiveness and completion
4. Participant engagement and learning

### Data to Collect
- Pre/post assessment scores
- Exercise completion rates
- Timing measurements per module
- Qualitative feedback comments

## Session 1: Technical Testing (Day 429)

### Preparation (15 minutes before start)
1. Verify all test environments are ready
2. Distribute pre-workshop survey
3. Test audio/video/screen sharing
4. Prepare backup materials and alternatives

### Welcome and Introduction (10 minutes)
- Welcome participants
- Explain testing objectives
- Review schedule and expectations
- Address any immediate technical issues

### Environment Setup Testing (30 minutes)
**Instructions to participants:**
"Please work through the pre-workshop setup checklist. Verbally report any issues you encounter. Don't solve them silently - we need to see what problems arise."

**Facilitator actions:**
- Observe without intervening initially
- Note common issues and timing
- After 20 minutes, provide targeted assistance
- Document all encountered issues

### Exercise Testing (60 minutes)
**Cycle for each exercise:**
1. Introduce exercise (2 minutes)
2. Participants attempt independently (8 minutes)
3. Group discussion of challenges (5 minutes)
4. Document issues and timing (2 minutes)

**Exercises to test:**
1. Basic pattern query (beginner)
2. Trajectory visualization (intermediate)
3. Infrastructure correlation (advanced)
4. Custom analysis script (expert)

### Wrap-up and Feedback (20 minutes)
- Distribute post-session survey
- Gather verbal feedback on technical aspects
- Review timing and pacing observations
- Schedule Session 2 reminders

## Session 2: Full Workshop Dry Run (Day 430)

### Preparation
- Review Session 1 feedback and adjustments
- Update materials based on identified issues
- Prepare timing tracking spreadsheet
- Assign observation roles if multiple facilitators

### Module Timing Targets
- **Module 1**: 30 minutes (Introduction)
- **Module 2**: 60 minutes (Python Client)
- **Module 3**: 60 minutes (Research Application)
- **Module 4**: 30 minutes (Next Steps)
- **Buffer**: 10 minutes (Q&A, transitions)

### Facilitation Techniques to Test
1. **Interactive polling**: Quick comprehension checks
2. **Think-pair-share**: Individual then small group discussion
3. **Live coding**: Demonstration with participant follow-along
4. **Breakout rooms**: Small group collaboration
5. **Gallery walk**: Sharing and feedback on group work

### Data Collection During Session
**Timing tracker:**
```
Module 1 - Start: __:__
  Introduction: __ minutes
  Framework overview: __ minutes
  Validation context: __ minutes
  Q&A: __ minutes
  End: __:__ (Target: 30 minutes, Actual: __ minutes)
```

**Engagement observations:**
- Number of participant questions per module
- Participation in interactive elements
- Body language and attention cues
- Technical issue frequency and type

### Post-Session Debrief
**Immediate feedback (15 minutes):**
- Quick round of "What worked well?"
- Quick round of "What needs improvement?"
- Specific technical issues encountered

**Detailed survey distribution:**
- Send post-workshop survey link
- Schedule follow-up survey for one week later
- Collect any additional written feedback

## Troubleshooting During Testing

### Common Issue Response Protocol
1. **Minor individual issue**: Provide quick solution, continue observing
2. **Common issue affecting multiple participants**: Pause, address group, document
3. **Major technical blocker**: Switch to backup plan, document thoroughly
4. **Content confusion**: Clarify, provide alternative explanation, note for revision

### Backup Plans
- **Network/API issues**: Switch to offline datasets and cached examples
- **Environment problems**: Provide pre-configured cloud environments
- **Time overruns**: Identify optional content to skip or shorten
- **Engagement drops**: Insert quick interactive activity or break

## Documentation Requirements

### Session 1 Documentation
- Technical issues log with frequency and severity
- Exercise completion rates and timing
- Environment setup success rate
- Participant technical proficiency observations

### Session 2 Documentation
- Module timing variances (planned vs actual)
- Engagement metrics (questions, participation)
- Content comprehension assessment
- Workshop flow and transition effectiveness

### Combined Analysis Report
- Priority issues ranked by impact and frequency
- Recommended changes to materials
- Adjusted timing recommendations
- Updated troubleshooting guide additions

## Success Criteria for Testing Completion

### Technical Requirements Met
- ≥90% of participants can complete all exercises
- All code examples execute without modification
- Environment setup succeeds for all test configurations
- API and data access works reliably

### Content Requirements Met
- All modules fit within time allocations (±10%)
- Participant comprehension scores improve ≥30%
- Engagement remains high throughout (≥80% active participation)
- Learning objectives are achieved per participant feedback

### Materials Requirements Met
- All documentation is accurate and accessible
- Troubleshooting guide covers all encountered issues
- Support materials are comprehensive and helpful
- Workshop can be delivered by other facilitators
